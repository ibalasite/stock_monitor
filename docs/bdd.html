§2 Schema 規則（TP-DB-001 ~ 005）

5 個 Schema 場景驗證 SQLite 所有 CHECK constraint、partial unique index 與格式規則， 確保資料層的業務約束在 migration 後立即生效。

Rule: Schema 與 migration 約束 5 Scenarios
TP-DB-001
watchlist cheap_price 不得大於 fair_price
Given 已完成資料庫 migration
When  新增 watchlist "2330" with fair 1500 and cheap 1000
Then  寫入應成功
When  新增 watchlist "2331" with fair 900 and cheap 1000
Then  寫入應失敗且錯誤為 CHECK constraint
TP-DB-002
valuation_methods 同 method_name 僅允許一個 enabled=1
When  插入 emily_composite:v1 with enabled 1 → 成功
When  插入 emily_composite:v2 with enabled 1 → partial unique index 失敗
TP-DB-003
message UNIQUE(stock_no, minute_bucket) + JSON 陣列格式
When  同分鐘 "2026-04-10 10:21" 插入第二筆 → unique constraint
When  minute_bucket 格式 "2026/04/10 10:22" → CHECK constraint
TP-DB-004
pending_delivery_ledger 可寫入並依狀態索引查詢
When  寫入 status "PENDING"
Then  應可成功查回且索引命中
TP-DB-005
valuation_snapshots 唯一鍵含 method_version（v1 ≠ v2）
v1 + v2 可共存；再插入 v1 → unique constraint

TP-ENV-003
LINE 必要參數缺失或無效時 fail-fast（Scenario Outline）
Examples: 規範命名缺失（LINE_*）
  missing LINE_CHANNEL_ACCESS_TOKEN → fail-fast
  missing LINE_TO_GROUP_ID          → fail-fast
Examples: 別名命名缺失（CHANNEL_*/TARGET_*）
  missing CHANNEL_ACCESS_TOKEN      → fail-fast
  missing TARGET_GROUP_ID           → fail-fast
Examples: 設定值無效
  invalid channel token             → fail-fast
  invalid group id                  → fail-fast

所有 case 均需確認 log 不得輸出完整 token 明文。

Rule: 股票名稱唯一來源為 DB（FR-18 TP-NAME-*） 3 Scenarios
TP-NAME-001 TWSE 與 Yahoo adapter 報價 dict 不含 name 欄位
  • Given TWSE adapter 回應包含 API 名稱 "台積電_API"
  • When 呼叫 TWSE get_realtime_quotes
  • Then TWSE 報價 dict 不應包含 "name" 欄位
  • Given Yahoo adapter 回應包含 API 名稱 "台積電_API"
  • When 呼叫 Yahoo get_realtime_quotes
  • Then Yahoo 報價 dict 不應包含 "name" 欄位
TP-NAME-002 觸發通知列使用 DB 股票名稱，不使用 API 報價名稱
  • Given DB watchlist "2330" 的 stock_name 為 "台積電_DB"
  • And 市場報價 "2330" 的 name 欄位為空字串
  • And 市場價 1900 跌破合理價 2000
  • When 執行分鐘輪詢
  • Then LINE 觸發通知應包含 "台積電_DB(2330)"
TP-NAME-003 只有 14:00 估值 job 才抓取並回存股票名稱到 DB
  • Given DB watchlist "2330" 的 stock_name 為空字串
  • And 行情來源對 "2330" 回傳名稱 "台積電"
  • When 在交易日 "14:00" 觸發估值（傳入 watchlist_repo 與 market_data_provider）
  • Then watchlist "2330" 的 stock_name 應被更新為 "台積電"
