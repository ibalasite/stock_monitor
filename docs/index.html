核心功能

從行情抓取到補償機制，每個環節都有對應的測試保障

📡

盤中即時監控

交易時段（09:00–13:30）每 60 秒抓取一次委賣一價格，與監控清單比對後觸發訊號。

🔗

雙行情來源

TWSE MIS（主）+ Yahoo Finance（副），以 tick_at 較新者為準（Freshness-First），Yahoo 失敗不中斷主流程。

🛡

5 分鐘冷卻機制

同 stock_no + stock_status 組合 5 分鐘內不重複推播，防止訊號疲勞，且不更新 update_time。

💬

LINE 模板化通知

所有 LINE 訊息（彙總/摘要/測試推播）均由 template_key + context 渲染，文案不硬編碼於業務程式。

📊

每日 14:00 估值

三種估值方法每交易日 14:00 自動逐股運算，結果寫入 SQLite 供隔日監控使用。

🔄

補償機制

LINE 推送成功 + DB 落盤失敗時，自動寫入 pending_delivery_ledger，補償完成前視同已通知，避免重複推送。

艾蜜莉複合法

股利法、歷年股價法、本益比法、淨值比法取平均，再乘安全邊際係數（0.9）。

fair = avg(子法合理價) × 0.9
oldbull_dividend_yield_v1

老牛殖利率法

使用平均股利與目標殖利率反推價格，簡單直觀，適合高股利穩定型股票。

fair = avg_dividend ÷ 5% cheap = avg_dividend ÷ 6%
raysky_blended_margin_v1

雷司紀四法融合

PE、股利、PB、NCAV 四法取中位數融合，套安全邊際係數（0.9）產生便宜價。

cheap = median(PE/Div/PB/NCAV) × 0.9

快速啟動

5 分鐘完成環境設定，開始盤中自動監控

1 · 安裝依賴
python -m pip install --require-hashes \
    -r requirements-dev.txt
2 · 設定 LINE 環境變數
$env:LINE_CHANNEL_ACCESS_TOKEN = "..."
$env:LINE_TO_GROUP_ID = "C..."
3 · 初始化 DB + 啟動
python -m stock_monitor \
    --db-path data/stock.db init-db
python -m stock_monitor \
    --db-path data/stock.db run-daemon
4 · 執行測試（CI gate）
python -m pytest -q tests
