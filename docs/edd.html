§2 關鍵業務規則（定版）

訊號狀態定義

status語意觸發條件
1低於合理價（below fair）market_price ≤ fair_price
2低於便宜價（below cheap）market_price ≤ cheap_price
若同時符合 status=1 與 2 → 只發 status=2（達便宜價必然達合理價，2 才是當下重點）。

多方法命中規則（§2.3）

  • 同分鐘同股票，多方法都符合 status=1 → 只產生一個股票層級訊號（status=1），訊息附命中方法清單。
  • 同分鐘同股票，有方法命中 status=1 且另有方法命中 status=2 → 統一產生 status=2，附完整命中方法清單。

不可變常數

項目值
時區Asia/Taipei（不接受 fallback UTC）
輪詢間隔60 秒
冷卻秒數300 秒（stock_no + stock_status）
交易時段09:00–13:30（含）
開盤檢查起始08:45
報價新鮮度門檻STALE_THRESHOLD_SEC=90
行情重試上限MAX_RETRY_COUNT=3
每分鐘 LINE 訊息數最多 1 封（彙總）
日結估值時間14:00
冪等鍵組成stock_no + minute_bucket（不含 stock_status）

§3.3 雙行情來源 Adapter 規格

圖 5 — Freshness-First 取捨流程：TWSE cache vs. Yahoo tick_at 比較

TwseRealtimeMarketDataProvider

欄位說明
端點https://mis.twse.com.tw/stock/api/getStockInfo.jsp
a 欄委賣五檔（_ 分隔）；a.split('_')[0] = 委賣一 = 系統 price
tlong毫秒時間戳 → tick_at = tlong // 1000
_price_cachea 有值時更新；a 為空或 - 時讀取 cache
冷啟動 seedcache 空且 a 無效時，以 y（昨收）種子填充
_exchange_cache由 ex 欄更新（tse 或 otc）

YahooFinanceMarketDataProvider

欄位說明
端點https://tw.stock.yahoo.com/quote/{stock_no}
priceHTML 委賣價區塊委賣一（去逗號）；盤後 fallback regularMarketPrice
tick_at"regularMarketTime":XXXX（unix seconds）
HTTP 失敗WARN log，回傳空 dict，不中斷主流程（CR-ADP-01）
exchange_map接受但不做 URL 建構用途（介面相容性）

CompositeMarketDataProvider（Freshness-First）

  • Rule 1twse && yahoo 均有值 → tick_at 較新者勝；相等時以 twse 為準。
  • Rule 2僅 twse 有值 → 使用 twse。
  • Rule 3僅 yahoo 有值（twse cache 空，冷啟動）→ 使用 yahoo。
  • Rule 4兩者皆無 → 不加入結果（觸發 STALE_QUOTE）。
CR-ADP-02：CompositeMarketDataProvider 禁止直接回傳任一來源字典而不做 Freshness-First 比較。

§4.1 盤中每分鐘流程

圖 3 — 盤中流程全圖：交易判斷 → 行情抓取 → 訊號評估 → 冷卻過濾 → 彙總發送 → 落盤

流程步驟

  1. Tick every 60s → 判斷是否交易時段
  2. 當日開盤摘要是否已發送 → 否則先發一封開盤摘要
  3. 載入啟用的 watchlist → 抓取所有股票委賣一報價
  4. 對每股票逐方法評估訊號（status=1 / status=2）
  5. 套用優先序（2 > 1）和冷卻過濾（stock_no + stock_status）
  6. 若無可發事件 → 結束；否則建立 render context
  7. 呼叫 render_line_template_message(template_key, context)
  8. 發送 LINE（每分鐘最多 1 封彙總）
  9. 發送成功 → DB transaction 一次提交所有 message rows
  10. 發送失敗 → 只寫 system_logs；DB 失敗 → 寫 pending_delivery_ledger

ColumnTypeKeyDefaultConstraint／說明
idINTEGERPKAUTOINCREMENT
stock_noTEXTFK—→ watchlist.stock_no

CREATE TABLE IF NOT EXISTS message (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  stock_no      TEXT    NOT NULL,
  message       TEXT    NOT NULL,
  stock_status  INTEGER NOT NULL CHECK (stock_status IN (1,2)),
  methods_hit   TEXT    NOT NULL
    CHECK (json_valid(methods_hit) AND json_type(methods_hit) = 'array'),
  minute_bucket TEXT    NOT NULL
    CHECK (
      length(minute_bucket) = 16
      AND minute_bucket GLOB '[0-9][0-9][0-9][0-9]-[0-9][0-9]-[0-9][0-9] [0-9][0-9]:[0-9][0-9]'
      AND substr(minute_bucket,5,1)  = '-'
      AND substr(minute_bucket,8,1)  = '-'
      AND substr(minute_bucket,11,1) = ' '
      AND substr(minute_bucket,14,1) = ':'
    ),
  update_time   INTEGER NOT NULL,
  FOREIGN KEY (stock_no) REFERENCES watchlist(stock_no),
  UNIQUE (stock_no, minute_bucket)
);
CREATE INDEX IF NOT EXISTS idx_message_cooldown
ON message(stock_no, stock_status, update_time DESC);

§7（擴充）文字檔模板設計（FR-17）

FR-14 既有規則已要求所有 LINE 訊息必須經 render_line_template_message(template_key, context) 渲染。 FR-17 在此之上要求渲染引擎可從外部純文字 .j2 檔讀取模板， 使非工程人員得以用記事本直接修改 wording，無須觸碰 Python 程式。

Clean Architecture 層次對映

Clean Architecture 層元件FR-17 職責
Interface Layer — 可透過 LINE_TEMPLATE_DIR 環境變數指定模板目錄
Application Layer render_line_template_message(key, ctx) 唯一呼叫入口（CR-ARCH-03）；只傳 context dict，不感知 Jinja2
Infrastructure Layer LineTemplateRenderer / TemplateRepository 持有 Jinja2 Environment，負責從 .j2 檔載入並渲染
模板檔案 templates/line/*.j2 純文字「View」層；企劃可直接編輯，不涉及 Python

MVC 類比

現行做法（FR-14 前）

# message_template.py — wording 硬寫在 .py return f"{label}目前{price}，\ 低於合理價{fair}"

企劃要改「合理價」→「Fair Value」需 Pull Request + Code Review。

FR-17（Jinja2 檔案載入）

# templates/line/trigger_row.j2 — 純文字 {{ display_label }}目前{{ current_price }}， {%- if stock_status == 2 %} 低於便宜價{{ cheap_price }}（合理價{{ fair_price }}） {%- else %} 低於合理價{{ fair_price }} {%- endif %}

企劃直接編輯 .j2 文字檔，不需動 Python。

工程實作規格

  • Jinja2 環境 Environment(loader=FileSystemLoader(template_dir), undefined=StrictUndefined, autoescape=False)；LINE 為純文字，不需 HTML autoescape；StrictUndefined 確保未定義變數立即報錯而非靜默空字串。
  • Key → 檔名 template_key 直接對應 {key}.j2；禁止含 /、\、..（路徑遍歷攻擊，OWASP A01）。實作建議白名單正規表達式 ^[a-z0-9_]+$。
  • 目錄覆蓋 預設目錄 templates/line/（相對 working directory）；可透過環境變數 LINE_TEMPLATE_DIR 覆蓋，方便測試與容器部署。
  • Fallback 若 .j2 檔不存在（TemplateNotFound），fallback 至 message_template.py 內嵌預設邏輯並寫入 WARN log（TEMPLATE_NOT_FOUND）；不得靜默降格或傳 None。
  • 渲染失敗 語法錯誤或 StrictUndefined 變數缺失導致渲染失敗 → 寫 ERROR log（TEMPLATE_RENDER_FAILED），不得以未知硬編碼格式送出。
  • 介面不變 render_line_template_message(template_key: str, context: dict) -> str 函式簽名永遠不變（CR-ARCH-03）；Application / Domain layer 零感知 Jinja2 存在。
  • 版本庫管理 templates/ 目錄隨原始碼一同提交 Git；wording 修改需提交 commit，但不觸碰 Python，不需 Code Review。
CR-ARCH-03 重申：render_line_template_message 只允許在 stock_monitor/application/message_template.py 中有唯一定義； 任何其他模組不得重複定義同名函式。

§13 Code Review 禁止清單（v1.0 定版）

下列規則一旦違反即視同 CR 不通過。業務程式禁止出現以下模式：
  • CR-SEC-01 LinePushClient 持有物件不得透過 repr() 或 log 輸出 token 明文；需用 field(repr=False) 保護。
  • CR-SEC-02 scenario_case 分支禁止存在於任何生產估值計算路徑。
  • CR-SEC-03 _resolve_timezone 對無效時區名稱禁止靜默 fallback UTC。
  • CR-ARCH-01 app.py（CLI Interface Layer）內禁止定義估值計算邏輯。
  • CR-ARCH-02 禁止在估值計算路徑中存在 scenario_case 分支（重申 CR-SEC-02）。
  • CR-ARCH-03 render_line_template_message 只允許在 message_template.py 中定義，禁止在其他模組重複定義。
  • CR-CODE-05 TimeBucketService 對無效時區名稱禁止靜默設 self._tz = None。
  • CR-ADP-01 YahooFinanceMarketDataProvider 在 HTTP 失敗時禁止 raise exception 向上傳播。
